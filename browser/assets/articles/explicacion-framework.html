Explicación del Funcionamiento del Framework Angular

5. Funcionamiento del Framework:

a. ¿Actúa sobre páginas, ficheros, secciones…?

Angular es un framework frontend que actúa sobre aplicaciones de una sola página (SPA - Single Page Application). Estas aplicaciones se cargan una vez en el navegador y manejan dinámicamente el contenido sin necesidad de recargar completamente la página. Angular interactúa con:

  • Componentes: Cada sección de la interfaz se define como un componente reutilizable.
  • Módulos: Agrupan funcionalidad y componentes relacionados.
  • Plantillas HTML: Definen el diseño visual que el usuario ve en la aplicación.
  • Estilos CSS: Aplican el diseño a los componentes.
  • Datos: Gestiona y enlaza dinámicamente los datos mediante el two-way data binding.

b. ¿Cómo se editan/modifican los ficheros que actúan en el framework?

Angular organiza los proyectos de forma modular, utilizando una estructura clara y estandarizada:

  1. Editar Componentes:
    • El archivo nombre-componente.component.ts contiene la lógica del componente.
    • El archivo nombre-componente.component.html define la estructura visual.
    • El archivo nombre-componente.component.css o .scss contiene los estilos específicos.

    Ejemplo de modificación en el archivo app.component.html:

    <h1>Bienvenido a Angular</h1>
<p>Este es un componente modificado.</p>
  2. Crear Nuevos Componentes: Usa el comando Angular CLI: 
    ng generate component nombre-componente
    Esto genera automáticamente los archivos necesarios para el nuevo componente.
  3. Modificar Módulos:

    Los módulos se definen en archivos nombre-modulo.module.ts, como app.module.ts. Aquí puedes importar y declarar componentes: 

    import { NombreComponente } from './nombre-componente/nombre-componente.component';

@NgModule({
  declarations: [
    NombreComponente,
    OtroComponente
  ],
  imports: [
    BrowserModule
  ],
  bootstrap: [AppComponent]
})
export class AppModule { }

c. ¿Cómo se referencian los ficheros creados desde otras aplicaciones (si es el caso)?

Angular facilita la interacción con archivos externos y APIs mediante varias estrategias:

  1. Importar archivos CSS o JavaScript: Puedes incluir archivos externos en el archivo angular.json: 
    {
  "styles": [
    "src/styles.css",
    "https://cdnjs.cloudflare.com/ajax/libs/bootstrap/5.2.0/css/bootstrap.min.css"
  ],
  "scripts": [
    "https://cdnjs.cloudflare.com/ajax/libs/jquery/3.6.0/jquery.min.js"
  ]
}
  2. Consumir APIs externas: Angular utiliza el módulo HttpClient para interactuar con APIs. Ejemplo de servicio: 
    import { Injectable } from '@angular/core';
import { HttpClient } from '@angular/common/http';

@Injectable({
  providedIn: 'root'
})
export class ApiService {
  private apiUrl = 'https://api.example.com/data';

  constructor(private http: HttpClient) { }

  getData() {
    return this.http.get(this.apiUrl);
  }
}
  3. Usar librerías externas: Instala librerías con npm e impórtalas en tu proyecto. Ejemplo: 
    npm install lodash
    Luego, utilízalas en tu componente: 
    import * as _ from 'lodash';

const result = _.chunk([1, 2, 3, 4], 2);
console.log(result); // [[1, 2], [3, 4]]
Nota: Asegúrate de mantener las dependencias y referencias organizadas en los archivos de configuración como angular.json y package.json.
