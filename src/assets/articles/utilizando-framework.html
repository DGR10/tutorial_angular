Utilización del Framework Angular

4. Utilización del Framework:

a. ¿Cómo empezar a crear una aplicación base?

  1. Crear un nuevo proyecto: Abre tu terminal o consola de comandos y ejecuta: 
    ng new nombre-aplicacion
    Este comando hará lo siguiente:
    • Crear una estructura básica de directorios y archivos.
    • Instalar dependencias necesarias con npm.
    • Configurar el proyecto con opciones interactivas como la inclusión de routing y el uso de SCSS, CSS, o LESS.
  2. Explorar la estructura del proyecto: Una vez creado, encontrarás las siguientes carpetas y archivos principales:
    • src/: Carpeta principal donde resides el código fuente de la aplicación.
    • app/: Contiene los componentes y lógica principal de la aplicación.
    • angular.json: Archivo de configuración del proyecto.
    • package.json: Lista de dependencias y scripts.
  3. Configurar el componente principal: Edita el archivo src/app/app.component.html para personalizar la vista principal. Por ejemplo: 
    <h1>¡Bienvenido a mi aplicación Angular!</h1>
    También puedes añadir un nuevo componente para organizar mejor tu aplicación: 
    ng generate component nombre-componente

b. ¿Cómo visualizar los resultados de la ejecución?

  1. Iniciar el servidor de desarrollo: Usa el siguiente comando para iniciar un servidor local: 
    ng serve
    Esto iniciará un servidor de desarrollo en el puerto predeterminado 4200.
  2. Acceder a la aplicación: Abre tu navegador y ve a la URL: 
    http://localhost:4200
    Aquí podrás visualizar la aplicación en funcionamiento.
  3. Actualizar cambios en tiempo real: Mientras el servidor está activo, cualquier cambio en los archivos fuente se reflejará automáticamente en el navegador.
  4. Visualizar errores y soluciones: Si encuentras errores durante la ejecución, verifica los logs en la terminal o en la consola del navegador.
Nota: Si deseas cambiar el puerto del servidor, puedes usar el comando: 
ng serve --port 4300
Esto ejecutará la aplicación en http://localhost:4300.

c. Extender tu aplicación:

  • Rutas: Configura la navegación entre páginas editando app-routing.module.ts.
  • Servicios: Genera servicios para gestionar lógica de negocio con: 
    ng generate service nombre-servicio
  • Conexión con APIs: Usa HttpClientModule para realizar llamadas a APIs externas.
